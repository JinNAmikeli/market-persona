# Agent Rules

状态：生效
版本：Market Harness Agent v0.1
更新日期：2026-07-08

## 1. 工作原则

后续所有 Agent 实现工作必须遵守：

- 先确认 Issue 范围，再改动文件。
- 优先保持现有架构和契约稳定。
- 小步修改，小步验证。
- 不把治理任务扩大成业务实现。
- 不把实现任务扩大成架构重构。
- 不回滚用户已有改动。

## 2. 允许做什么

在有明确 Issue 或用户指令时，允许：

- 修改对应文档。
- 修复与 Issue 直接相关的 bug。
- 补齐 schema、测试或验证脚本。
- 增强现有 Runtime 环节。
- 增强 trace、memory、wiki、reflection 的可审计性。
- 更新 README 或 Runbook，使其和实际行为一致。

## 3. 禁止做什么

禁止：

- 在治理文档任务中修改业务代码。
- 在未批准时新增功能。
- 在未批准时重构目录或模块边界。
- 删除或重命名冻结 API。
- 提交 `.env`、密钥、trace、memory 或其他私有运行数据。
- 绕过 Reflection 返回金融建议。
- 生成买入、卖出、目标价、确定收益等内容。
- 把模板 fallback、LLM fallback 或数据缺失伪装成完整证据。

## 4. 需要用户批准

触及以下事项必须先获得用户批准：

- 合规边界调整。
- 投资表达模板变化。
- 主动触达策略。
- Wiki 内容审核口径。
- 外部数据源或外部服务。
- 存储迁移。
- 多用户权限系统。
- 真实交易、账户、组合、仓位相关功能。
- 破坏性 API 或 schema 变更。

## 5. 金融回答规则

Agent 回答金融问题时必须：

- 说明数据时效。
- 区分事实、观察、推断和风险。
- 用观察指标替代操作建议。
- 对个股问题说明来源与边界。
- 在必要时给出标准免责声明。

推荐转写：

```text
我不能给出买卖指令，但可以帮你拆解这个标的/板块当前的热度来源、风险点和后续观察指标。
```

## 6. 修改后检查

实现类修改完成后，至少检查：

- `git diff --stat`
- 是否只改了 Issue 允许的文件。
- 是否需要运行 `python scripts/verify_runtime.py`。
- 是否需要更新 `docs/HANDOFF.md`。
- 是否新增了运行期文件，需要确认不会提交。
